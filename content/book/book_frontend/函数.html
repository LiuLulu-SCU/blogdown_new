26 函数

函数（function)：是被设计为执行特定任务的代码块

为什么需要函数？

函数有利于精简代码方便复用，提高开发效率。

例子：99乘法表，页面需要打印多个怎么办？99乘法表案例见 25.3.3。我们这里吧案例代码封装进一个函数中，方便后面调用。封装后的代码见这里。

26.1 函数的声明和调用

包括函数的声明和调用。

26.1.1 函数的声明

声明语法：声明关键词：function

function 函数名() {
  函数体
}

//例如
function sayHi() {
  document.write(`hai~~`)
}

26.1.2 函数体

函数体是函数的构成部分，它负责将代码“包裹”起来，直到函数调用时函数体内的代码才会被执行。函 数的功能代码都要写在函数体当中。

function sayHi() {
  document.write(`hai~~`) //函数体
}

26.1.3 函数的调用

声明（定义）的函数必须调用才会真正被执行，使用 () 调用函数

调用的语法

//函数调用，这些函数体内的代码逻辑会被执行
函数名()

26.2 函数的命名规范

  • 和变量命名基本一致，见22.3节。
  • 尽量小驼峰式命名法
  • 前缀应该为动词
//函数命名例子
function getName(){}
function addSquares(){}
表8.1: JS函数名-常用动词约定
情况 说明
can 判断是否可执行某个动作
has 判断是否含有某个值
is 判断是否没某个值
get 获取某个值
set 设置某个值

26.3 函数传参

26.3.1 声明语法

声明语法

function 函数名(参数列表) {
  函数体
}

//例子1：单个参数
function getSquare(num1) {
  document.write{num1 * num1}
}
//例子2：多个参数
function getSum(num1, num2) {
  document.write{num1 + num2}
}

参数列表

  • 定义：是传入函数的参数的列表
  • 作用：声明这个函数要传入数据
  • 写法：当传入多个参数时，参数间用逗号隔开

26.3.2 调用语法

调用语法

函数名(参数列表) 

//例子1：单个参数
getSquare(8) 
//例子2：多个参数
getSum(10, 20)

我们曾经使用过的 alert(‘打印’), parseInt(‘11’), Number(‘11’) 本质上都是函数调用的传参。

26.3.3 形参和实参

  • 形参：声明函数时写在函数名右边小括号里的叫形参（形式上的参数）
  • 实参：调用函数时写在函数名右边小括号里的叫实参（实际上的参数）

开发中尽量保持形参和实参个数一致。

function getSum(num1, num2) { //num1，num2是形参
  document.write{num1 + num2}
}

getSum(10, 20) //10,20是实参

形参可以理解为是在这个函数内声明的变量（let num1），实参可以理解为是给这个变量赋值（num1 = 10）。

26.3.4 参数默认值

形参： 可以看做变量，但是如果一个变量不给值，默认是什么？ undefined 但是如果做用户不输入实参，刚才的案例，则出现 undefined + undefined 结果是什么？NaN

我们可以改进下，用户不输入实参，可以给 形参默认值，可以默认为 0, 这样程序更严谨，可以如下操作：


function getSum(num1 = 0, num2 = 10) { //num1，num2是形参
  document.write{num1 + num2}
}

getSum() //结果是0，那不是NaN
getSum(10, 20) //10,20是实参

说明：默认值只会在缺少实参传递时才会被执行，所以有实参传递时会优先执行传递过来的实参。

注：也可以通过逻辑中断来实现参数默认值，参见第26.8章。

26.4 函数返回值

当调用某个函数，这个函数会返回一个结果出来，这就是有返回值的函数。

具有返回值的函数

无返回值的函数

返回值的语法

当函数需要返回数据出去时，用return关键字

function 函数名() {
  函数体
  return 返回值
}

//例子
function getSum(num1, num2) { 
  return num1 + num2
}

let num = getSum(10, 20)
doucument.write(num)

细节：

  • 在函数体中使用 return 关键字能将内部的执行结果交给函数外部使用
  • return 后面代码不会再被执行，会立即结束当前函数，所以 return 后面的数据不要换行写
  • return函数可以没有 return，这种情况函数默认返回值为 undefined

26.5 函数的一些细节

  • 两个相同的函数后面的会覆盖前面的函数
  • 在Javascript中 实参的个数和形参的个数可以不一致
    • 如果形参过多 会自动填上undefined

  • 函数一旦碰到return就不会在往下执行了 函数的结束用return

26.6 作用域

通常来说，一段程序代码中所用到的名字并不总是有效和可用的，而限定这个名字的可用性的代码范围就是这个名 字的作用域。 作用域的使用提高了程序逻辑的局部性，增强了程序的可靠性，减少了名字冲突。

  • 全局作用域：作用于所有代码执行的环境(整个 script 标签内部)或者一个独立的 js 文件
  • 局部作用域：作用于函数内的代码环境，就是局部作用域。 因为跟函数有关系，所以也称为函数作用域。

在JavaScript中，根据作用域的不同，变量可以分为：

  • 全局变量：函数外部let 的变量，全局变量在任何区域都可以访问和修改
  • 局部变量：函数内部let 的变量，局部变量只能在当前函数内部访问和修改
<script>
    //1.全局变量
    let num = 10 //全局变量
    console.log(num)

    //2.局部变量
    function fun() {
        let str = 'pink' //局部变量
    }
    console.log(str) //这里会报错，因为全局作用域中找不到，局部变量'pink'
</script>

注意：

  • 如果函数内部，变量没有声明，直接赋值，也当全局变量看，但是强烈不推荐
  • 函数内部的形参可以看做是局部变量
<script>
    let num = 20

     function fn() {
         num = 10 //函数内部的num变成了全局变量，会覆盖20
     }

    fn() //调用函数
    console.log(num) //10
</script>

26.6.1 变量的访问原则

访问原则：在能够访问到的情况下先局部，局部没有在找全局

作用域链：如果有嵌套函数时，采取就近原则的方式来查找变量最终的值

26.7 匿名函数

函数按是否有名字可以分为：具名函数和匿名函数。

匿名函数没有名字的函数, 无法直接使用，其使用方式有两种

  • 函数表达式
  • 立即执行函数

26.7.1 函数表达式

将匿名函数赋值给一个变量，并且通过变量名称进行调用，我们将这个称为函数表达式

//语法
let fn = function () {
  //函数体
}
fn()

//例子
let fn = function () {
    console.log('我是函数表达式。');
}
fn()

26.7.2 立即执行函数

场景介绍: 避免全局变量之间的污染

//语法
//方式1
(function () {函数体})(实参);
//方式2
(function () {函数体}(实参));

//例子
//方式1
(function (num1, num2) {
    console.log(num1 + num2);
})(1, 2); //3
//方式2
(function (num1, num2) {
     console.log(num1 + num2)
}(3, 4)); //7

注意： 多个立即执行函数要用 ; 隔开，要不然会报错

26.8 逻辑中断

使用情景：替换默认参数。

开发中，还会见到以下的写法：

function getSum1(num1, num2) {
    num1 = num1 || 0 //逻辑中断
    num2 = num2 || 0
    console.log(num1 + num2);
}
getSum1(1, 2) //3    
getSum1() //0

26.8.1 逻辑运算符里的短路

短路：只存在于 && 和 || 中，当满足一定条件时，就只判断/执行左侧代码，不判断/执行右侧代码。原因：通过左边能得到整个式子的结果，因此没必要再判断右边。一般用在变量赋值。

表26.1: JS逻辑运算符里的短路
符号 短路条件
&& 左边为false就短路
|| 左边为true就短路 
//逻辑与&& 前面为假，就不判断后面
console.log(false && 20) //false
console.log(5 < 3 && 20) //false
console.log(undefined && 20) //false
console.log(null && 20) //false
console.log(0 && 20) //0。0判断为flase，不再判断后面，直接返回0
console.log(10 && 20) //20

//逻辑或|| 前面为假，还需要判断后面
console.log(false || 20) //20
console.log(5 < 3 || 20) //20
console.log(undefined || 20) //20
console.log(null || 20) //20
console.log(0 || 20) //20。0判断为flase，还需判断后面20为true，返回20
console.log(10 || 20) //10
